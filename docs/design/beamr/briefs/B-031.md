# B-031: gleam_erlang_ffi — process utilities

> **Cluster:** beamr
> **Depends on:** B-027, B-030
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer
> **Stories:**
> - S2 (Workflow Author) — As a workflow author, I want each workflow step to run in its own isolated process.
> - S6 (Workflow Author) — As a workflow author, I want to define supervision trees.

## Purpose

gleam_erlang_ffi contains process utility functions beyond the selector system: `trap_exits/1`, `flush_messages/0`, `sleep/1`, `sleep_forever/0`, `demonitor/1`, `link/1`, `process_named/1`, `register_process/2`, `unregister_process/1`, `pid_from_dynamic/1`. These are thin wrappers around erlang BIFs we already have — most delegate directly. Implement as native Rust BIFs registered under `gleam_erlang_ffi` MFAs.

## Task

Add process utility functions to `crates/beamr/src/native/selector_ffi.rs` (or a new `gleam_ffi.rs`). These are simple delegations to existing facilities — trap_exits calls process_flag(trap_exit, true), link delegates to the LinkFacility, register/unregister delegate to RegistryFacility.

## Boundaries

- This brief SHALL NOT implement gleam_erlang_ffi:application_start (OTP application controller)
- This brief SHALL NOT implement node/cluster operations (net_kernel:connect_node)
- This brief SHALL NOT implement OS environment functions (os:getenv, os:putenv)

## Requirements

### R1: Process flag and link wrappers

Implement:
- `trap_exits/1`: Accept a boolean. Call process_flag(trap_exit, Bool) via LinkFacility. Return `nil`.
- `link/1`: Accept a Pid. Establish a bidirectional link via LinkFacility. Return `nil`.
- `demonitor/1`: Accept a reference. Remove the monitor via SupervisionFacility. Return `nil`.

These are thin wrappers that translate gleam_erlang's calling convention to the existing facility traits.

**Acceptance:**
- trap_exits(true) enables trap_exit on the calling process
- link(Pid) establishes a link
- demonitor(Ref) removes a monitor
- All return nil (Gleam's Nil type)

**Files:**
- modify: crates/beamr/src/native/selector_ffi.rs

**Checklist:**
- C150

### R2: Sleep, flush, and process registry wrappers

Implement:
- `sleep/1`: Accept milliseconds integer. Call timer:sleep. Return `nil`.
- `sleep_forever/0`: Block the process indefinitely (suspend with no timeout). Return never.
- `flush_messages/0`: Drain the process mailbox. Return `nil`.
- `register_process/2`: Accept (Name, Pid). Delegate to RegistryFacility. Return `nil`.
- `unregister_process/1`: Accept Name. Delegate to RegistryFacility. Return `nil`.
- `process_named/1`: Accept Name. Delegate to RegistryFacility (whereis). Return `{ok, Pid}` or `{error, nil}`.

**Acceptance:**
- sleep(100) suspends ~100ms
- flush_messages clears the mailbox
- register_process/unregister_process manage names
- process_named returns Ok/Error result tuple

**Files:**
- modify: crates/beamr/src/native/selector_ffi.rs

**Checklist:**
- C150

### R3: pid_from_dynamic/1 and registration

`pid_from_dynamic/1` SHALL accept a dynamic term and return `{ok, Pid}` if the term is a Pid, or `{error, [DecodeError]}` if not. Register all gleam_erlang_ffi functions in `register_gleam_erlang_ffi()`.

**Acceptance:**
- pid_from_dynamic(Pid) returns {ok, Pid}
- pid_from_dynamic(not_pid) returns {error, ...}
- All functions registered under gleam_erlang_ffi module
- Registration wired from CLI initialization

**Files:**
- modify: crates/beamr/src/native/selector_ffi.rs
- modify: crates/beamr-cli/src/main.rs

**Checklist:**
- C150
