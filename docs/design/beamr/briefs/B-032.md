# B-032: gleam_otp supervisor support — gleam_otp_external + end-to-end test

> **Cluster:** beamr
> **Depends on:** B-030, B-031
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer
> **Stories:**
> - S6 (Workflow Author) — As a workflow author, I want to define supervision trees over my workflow steps so that failed steps are automatically restarted.
> - S8 (Meridian Runtime) — As the Meridian runtime, I want to create a beamr VM instance, load .beam modules, and spawn processes.

## Purpose

gleam_otp's supervisor module calls `gleam_otp_external:application_stopped/0` and indirectly uses `supervisor:start_link/2`. Rather than implementing the full OTP supervisor module, gleam_otp's supervisor is itself a Gleam-level abstraction that uses spawn_link, monitor, trap_exit, and the actor message loop. The only external dependencies are `gleam_otp_external` functions and a few remaining BIFs. This brief implements those stubs and proves the full supervision chain works end-to-end.

## Task

Implement `gleam_otp_external` functions as native Rust BIFs. Write an end-to-end integration test that loads the compiled gleam_otp .beam modules, starts a supervisor with a child worker, verifies the child runs, crashes the child, and verifies the supervisor restarts it.

## Boundaries

- This brief SHALL NOT implement the full OTP supervisor module (supervisor:start_link/2 is stubbed)
- This brief SHALL NOT implement OTP application controllers
- This brief SHALL NOT implement hot code upgrade or code_change callbacks

## Requirements

### R1: gleam_otp_external stubs

Implement and register under `gleam_otp_external` module:
- `application_stopped/0`: Return `ok`. This is called when a supervised application shuts down. For beamr, it's a no-op.

Implement and register under `supervisor` module:
- `supervisor:start_link/2`: This is called by `gleam_otp_external`. For beamr, stub it to return `{ok, self()}` — gleam_otp's supervisor.gleam doesn't actually use the OTP supervisor module's return value for process management, it manages children directly via spawn_link and monitor.

Also register any remaining BIFs discovered during integration testing (erlang:++/2 for list append, erlang:length/1, etc.).

**Acceptance:**
- application_stopped() returns ok
- supervisor:start_link/2 returns {ok, Pid}
- Registration under correct module atoms
- No unresolved imports when loading all gleam_otp modules together

**Files:**
- create: crates/beamr/src/native/otp_stubs.rs
- modify: crates/beamr/src/native/mod.rs
- modify: crates/beamr-cli/src/main.rs

**Checklist:**
- C150

### R2: Remaining BIF gaps

During integration testing, identify and implement any remaining erlang BIFs that block execution. Likely candidates from the import analysis:
- `erlang:++/2`: List append
- `erlang:length/1`: List length
- `erlang:pid_to_list/1`: PID to string
- `erlang:get/0`: Process dictionary (return empty list)
- `erlang:send/2`: Already registered but may need wiring to actual mailbox delivery

Implement each as a native BIF in gate3_bifs or a new file.

**Acceptance:**
- All BIFs needed by gleam_otp execute correctly
- No unresolved import errors when running the full gleam_otp module set
- Tests cover each new BIF

**Files:**
- modify: crates/beamr/src/native/gate3_bifs/mod.rs

**Checklist:**
- C150

### R3: End-to-end supervisor integration test

Write a Gleam test program that:
1. Defines a simple worker actor (counter that responds to Ping)
2. Wraps it in a supervisor with one_for_one strategy
3. Starts the supervisor
4. Sends a message to the worker, verifies response
5. Kills the worker, verifies supervisor restarts it
6. Sends another message to the new worker, verifies response

Compile to .beam, load all modules (gleam_erlang, gleam_otp, gleam_stdlib, the test module), and execute via the scheduler.

**Acceptance:**
- Supervisor starts successfully
- Worker receives and processes messages
- Worker crash triggers supervisor restart
- Restarted worker handles messages correctly
- Test runs on the beamr scheduler (not single-process)

**Files:**
- create: crates/beamr/tests/supervisor_e2e.rs
- create: crates/beamr/tests/fixtures/otp_e2e/ (compiled test .beam files)

**Checklist:**
- C150
