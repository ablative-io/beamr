# beamr — Architecture Review

*State as of v0.12.0 (main @ 58987bb, July 2026).*

## What it is

A ground-up BEAM VM in Rust that loads OTP-26 `.beam` bytecode produced by the
Gleam toolchain (Gleam → Erlang source → `erlc` → `.beam`) and executes it
with preemptive scheduling, per-process isolation, supervision primitives, and
a native function interface. Four workspace crates:

- **`beamr`** — the VM (~100K lines). `no_std`-capable.
- **`beamr-cli`** — runner; also `record`/`replay`/`imports`/`compile` commands.
- **`beamr-wasm`** — browser bindings over a single-threaded cooperative scheduler.
- **`gleam-types`** — extracts Gleam type sidecars consumed by JIT specialization.

The feature matrix is the architectural split that matters: `threads` (native
N-thread work-stealing scheduler) vs `cooperative` (wasm `WasmScheduler`),
plus `net`, `fs`, `jit`, `embedded`, `json`, `telemetry`. The crate root
re-exports a stable actor surface (`Actor`, `ActorContext`, `spawn_actor`,
`spawn_actor_cooperative`, `DynActor`, `NativeHandler`…) on both runtimes so
downstream consumers never touch scheduler internals.

## Design discipline

Eleven ADRs (docs/adr/) pin the big choices. The ones that shape everything:

- **Execution model only, not ERTS.** No reimplementation of Erlang's full
  runtime system; only what Gleam-generated bytecode needs.
- **Gleam-only opcodes (ADR-005).** ~40–60 of ~170 BEAM opcodes; unknown
  opcode is a loud, named error.
- **Demand-driven BIFs (ADR-006) — "the import-table leash."** A BIF is
  implemented only when a real workflow's unresolved-import report demands it.
  This is the project's core scope-control discipline.
- **Supervision is library code (ADR-007).** The VM provides exactly four
  primitives — links, monitors, exit signals, trap_exit. Strategies live in
  `gleam_otp` as ordinary Gleam code.
- **No async in the scheduler hot path (ADR-003).** Plain OS threads +
  lock-free work stealing; tokio only at the edges.
- **Messages copy terms (ADR-008).** Deep-copy into the receiver's heap for
  total isolation; binaries >64 bytes share an off-heap refcounted Arc.
- **"The one rule."** beamr depends on nothing downstream; everything points
  inward. External operations plug in via the native boundary (NIF registry)
  and the reduction-boundary hook (ADR-009, a registration seam that is
  zero-cost when unregistered).

`docs/terminology.md` pins the vocabulary (reduction = budget unit not event;
boxed ≠ Rust `Box`; BIF = VM's job, NIF = embedder's job) and is worth
holding people to — the docs use it precisely.

## How execution actually works

**Terms.** One 64-bit word, low 3 bits are the tag. Immediates (small ints
±2^60, atoms, pids, nil) decode with a mask+shift; boxed terms are tagged
pointers to `[header, payload…]` heap objects; cons cells are headerless
2-word pairs. Binaries have three forms: inline (≤64 bytes, packed words),
ProcBin (>64 bytes, off-heap `Arc<Vec<u8>>` with manually managed refcounts),
and SubBinary (zero-copy view). Maps are flatmaps with sorted keys and linear
lookup — no hashmap variant exists.

**Interpreter.** Dispatches a pre-decoded 129-variant `Instruction` enum (no
runtime bytecode decoding). Reductions (budget 4000) are charged **only at
call edges** — straight-line code between calls is never preempted. Call
frames live in a Rust `Vec<StackFrame>`, each pinning an `Arc<Module>`; that
pinning is what makes dual-version hot code loading safe (`purge_old` refuses
while old code is referenced). Exceptions are a side stack of handlers; BIFs
route through a single seam (`call_native_entry`) that handles capability
audit, dirty-pool dispatch, replay interception, suspension, and trampolines
(BIFs like `lists:map` that must re-enter the interpreter to call a closure).

**GC.** Per-process generational copying collector, no global locks. Two bump
regions (nursery + old gen) whose backing storage never grows while live
pointers exist. The root set is an *ordered* contract: `roots()` and
`replace_roots()` must iterate identically — adding a rooted field means
editing both. Off-heap ProcBin bytes count toward a virtual binary heap that
forces collection pressure.

**Scheduler.** Per-thread run queues (four priority lanes, LIFO owner pops,
FIFO steals, bounded low-priority starvation). The central mechanism is the
**process slot protocol**: `Present(process)` ↔ `Executing(metadata shadow)`.
While a process runs a slice, side threads deliver messages/exit signals into
the shadow's pending buffers, merged back at slice end. Suspensions
(host-await, dirty call, hook) are **identity-gated by per-process monotonic
call ids** — stale completions drop, and a message arrival never wakes a gated
suspension (that would double-submit the parked side effect). Dirty CPU/IO
pools take long-running natives off the fair schedulers. Timer wheel ticks on
scheduler thread 0; receive-timeouts are marked cross-thread but applied only
by the owning thread.

**Native layer.** BIFs and NIFs are plain data tables of
`fn(&[Term], &mut ProcessContext) -> Result<Term, Term>` in twin DashMap
registries (BIF wins name collisions). Native processes are ordinary
`Process` structs carrying an optional Rust handler + restart factory — same
heap, mailbox, links, tombstones as bytecode processes; only the slice body
differs. The gen_server-style `Actor` facade sits on top. All real I/O goes
through a completion ring and suspends the process; nothing blocks a
scheduler thread.

**Hot code loading.** Registry holds current + one old version per module
name with monotonic generations. Closures survive reload via a deterministic
hash of module+function names. JIT cache keys include generation, so stale
compiled code naturally misses after a reload.

## Invariants a developer must not break

1. **BIF args are not GC roots.** A BIF holding a heap term across any
   allocation must park it in an x-register (or `with_rooted`) and re-read
   after. This is the #1 BIF-authoring trap.
2. **`roots()`/`replace_roots()` iteration-order coupling** — edit both or
   minor GC silently corrupts registers.
3. **`ConstantPool::clone` rebases interior pointers per boxed layout.** Any
   new pointer-bearing `BoxedTag` must be added to
   `rebase_boxed_block_terms` or cloned pools carry dangling pointers.
4. **NIL is the exact word `0b011`** — its tag value otherwise aliases boxed
   pointers. Never test nil by tag alone.
5. **Live-count discipline**: opcodes/BIFs calling `ensure_space` with an
   undercounted `live` value cause minor GC to NIL live x-registers.
6. **`Process::clone` drops native handlers** (deliberate); live native
   processes must never be cloned — supervisors restart via factory.
7. **Never call wake/deliver while holding a process-slot lock** (deadlock);
   the codebase drops locks first, consistently.
8. **Actor pids cross process boundaries as i64 scalars**, never pid terms —
   ETF delivery to an Executing receiver corrupts local pids.
9. **`SenderHandle::call` deadlocks inside an actor handler** (it blocks the
   worker thread); intra-actor request/reply must use cast + explicit reply.
10. **Infallible term constructors truncate in release** (`small_int`, `pid`);
    untrusted values must go through `try_*` + bignum promotion.
11. **The loader is DoS-hardened by budget-charging every allocation** —
    new chunk/ETF decoders must charge the budget or they reopen zip-bomb
    holes the tests guard against.

## Conventions and process

- Gate bar per brief: `fmt --check`, `check`, full test suite,
  `clippy --all-targets -- -D warnings`, no file >500 code lines (400 for
  entrypoints), no `unwrap()`/`expect()` outside tests, no bytecode-path
  regressions, adversarial review on concurrency-delicate work.
- Execution tracker: `docs/design/beamr/` — DESIGN/CHECKLIST/USER-STORIES/
  RUNBOOK + 177 briefs (B-130+ json-only = in-flight).
- Test fixtures are pre-compiled `.beam` binaries committed beside their
  `.erl`/`.gleam` sources; no in-repo compile step. 36 integration test files
  plus a 24-module Gleam-stdlib gate with a golden stdout file.
- No CI in-repo; gates run locally per RELEASE_CHECKLIST. Release via
  `scripts/release.sh` in dependency order (gleam-types → beamr → beamr-cli →
  beamr-wasm), tag only after explicit lead approval.
